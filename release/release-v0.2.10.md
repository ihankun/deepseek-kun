# Kun v0.2.10

这一版在 v0.2.9 的大更新之后做收尾打磨：Skill 可以按需开关、模型端点支持自定义、模型选择器更精简，同时把运行时解析和 Git 目录识别做得更健壮。

### Skill 管理与自定义端点

- Skill 新增管理开关，可以按需启用 / 停用，而不是一刀切。
- 支持自定义模型端点，接入更多自建或第三方服务更方便。

### 模型选择器更精简

- 在选择器里隐藏供应商的别名模型，避免同一个模型出现多个入口。
- 压缩了供应商模型能力徽标的展示，列表更紧凑清晰。

### Write 与编辑器

- Write 新增“引用选区”动作，选中文字可以快速作为引用带入。
- Write 导出菜单保持右对齐，编辑器选择器图标解析更准确，设置项操作也统一靠右。

### 运行时与 Git 健壮性

- 运行时探测会尝试备用 Kun 端口，连接更不容易因端口占用而失败。
- 强化了运行时的解析与错误处理。
- 无论 Git 语言环境如何，都能正确识别“非仓库目录”，避免误判。
- 失败的工具详情可以折叠，附件选择器的文件路径也通过 Kun bridge 暴露出来。

### 升级说明

- 从 `v0.2.9` 升级可直接通过 GUI 更新，无需额外操作。

### 总结

v0.2.10 是 v0.2.9 之后的一轮精修：Skill 开关、自定义端点、更干净的模型选择器，加上运行时和 Git 识别的健壮性提升，让大更新落地得更稳。
