# Kun v0.2.11

这一版带来三块面向“认真干活”的能力：为并行 Agent 准备的 Git worktree 池、用于代码导航的 LSP 工具，以及独立的记忆管理面板。同时 MCP、插件和 Kun 运行时也各有补强。

### 并行 Agent 与 Worktree 池

- 新增 Git worktree 池管理，让多个 Agent 可以在各自隔离的工作树里并行干活。
- 加固 worktree 操作：防止分支被劫持、减少破坏性清理、修正 stash 警告。

### LSP 代码导航工具

- 新增 LSP AI 工具，支持 TS / JS 的代码导航（定义、引用等）。
- 加固 LSP 进程管理：避免遗留孤儿进程、修复 acquire 竞态、崩溃时正确 reject。

### 记忆管理

- 设置里新增独立的记忆管理面板，并提供启用 / 停用开关。
- 用户级记忆始终注入，并修复了中日韩文（CJK）记忆的检索问题。
- 保存失败时编辑器保持打开并给出停用警告，避免内容丢失。

### MCP 与插件

- 扩展市场支持 MCP 服务器开关，按需启停更方便。
- MCP 插件安装后保留目录描述；移除了文件系统类的默认推荐。

### Kun 运行时与供应商

- 省 token 模式不再改写历史，避免上下文被悄悄改动。
- 中断时保留已生成的部分响应，不再整段丢弃。
- composer 的模型选择器只展示已配置供应商的模型；切换图片供应商不再清空自定义 `baseUrl` / `apiKey`。
- Anthropic 消息请求会带上 effort 参数。

### 构建

- Windows 发布包内置多尺寸 `.ico`，小图标在资源管理器和桌面上更清晰。

### 升级说明

- 从 `v0.2.10` 升级可直接通过 GUI 更新。
- 如果你要并行跑多个 Agent，可以在设置里了解新的 worktree 池；用到记忆功能的话，留意新的记忆管理面板和开关。

### 总结

v0.2.11 是一次“生产力工具”加强版：worktree 池、LSP 导航和记忆管理三件套，配合 MCP、插件与运行时的补强，让 Kun 更适合承接真实、并行的开发工作。
