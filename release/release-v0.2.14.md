# Kun v0.2.14

这是一次内容极其丰富的大版本，也是近两周开发成果的一次集中发布。两大主线是：让视觉模型能够真正“操控桌面”的 Computer Use，以及对上下文压缩（compaction）的全面重做。围绕这两条线，还带来了证据驱动的研究简报、设计质量检查、worktree 融入会话生命周期，以及大量界面与稳定性改进。从这一版起，应用更新后还会直接展示更新说明。

### Computer Use：让视觉模型操控桌面

- 新增 Computer Use 能力，视觉模型可以通过截屏 + 鼠标键盘操控真实桌面。
- 读图结果会注入给视觉模型（#375），截图作为 `tool_result` 的兄弟节点发出以兼容 Anthropic 协议。
- 截图不再把上下文容量表撑爆，瞬时网关错误会自动重试。
- macOS 体验打磨：主动把 Kun 登记进系统权限列表、显示“已授予辅助功能，需重启生效”状态、隐藏 kun-child 的 Dock 图标（不再出现两个 Kun 图标）。

### 上下文压缩全面重做

- 压缩摘要重新排序，渲染在每个回合的末尾，阅读更连贯。
- 支持模型生成的压缩摘要，并实现“有效历史”管理，压缩后上下文保留更合理。
- 引入 prompt token 信任因子，应对部分供应商把缓存读数折进 prompt_tokens 造成的虚高。
- 手动 `/compact` 会作为一行进度出现在对话中；未知模型的上下文默认按 128k 处理。

### 研究、设计与需求

- 新增证据驱动的研究简报（#386），研究产出更有出处。
- 内置设计质量检查器与配套的上下文管理。
- 把 PM 技能框架接进新建需求工作流，需求阶段更有方法。

### 工作区与窗口

- worktree 池融入会话生命周期（#365），并行 Agent 与会话的绑定更自然。
- 新增工作区文件树预览面板（#356）。
- 归档会话移入设置（#353）；新增关闭行为提示（#354）与光标聚光灯偏好（#393）。

### 供应商、更新与目标续跑

- 供应商设置解除保存阻塞并支持批量模型管理（#397 #399）；新增带作用域的模型代理设置。
- 更新后展示更新说明（#388）；加固 macOS 更新启动恢复。
- 中断后自动续跑进行中的目标（#370），并恢复空的工具后续生成（#387）。

### 其他改进与修复

- MCP 服务器后台自动重连，并增强文件提及处理；补充主流 MCP 推荐（#351）。
- 修复 streamdown 重复文本（#381）、跨平台托盘图标缩放（#384）、保留 Skill 停用偏好（#394）。
- 每个会话独立保存模型选择；Write 工具栏显示字数（#305）；飞书 / Lark 流式输出处理改进（#362）。

### 升级说明

- 从 `v0.2.13` 升级可直接通过 GUI 更新，更新完成后会自动展示本篇更新说明。
- 想用 Computer Use 的话，macOS 上需要按提示授予“辅助功能 / 屏幕录制”权限，并按提示重启一次。

### 总结

v0.2.14 是 v0.2 系列里集大成的一版：Computer Use 让模型走出聊天框、操控真实桌面，压缩重做让长会话更经得起跑，研究简报、设计质量、worktree 生命周期和大量修复则让整体能力更完整、更耐用。
