# Kun v0.2.15

这是 v0.2 系列里功能跨度最大的一版。主角是全新的可视化工作流「创建 Loop」——一个 n8n 与 dify 风格融合的节点式编排画布，让“把多步 Agent 流程画出来、跑起来、复用起来”成为现实。围绕它，这一版还带来了按供应商路由、对话分叉、回合回滚检查点、会话级 worktree 等一批面向真实工作流的能力，并合入了一轮针对输入框、侧栏、供应商和压缩的修复。

### 可视化工作流「创建 Loop」

- 在定时任务下方新增 n8n 风格的可视化工作流编辑器：全屏聚焦画布、节点拖拽连线、按节点单独执行。
- 丰富的节点体系：触发器（手动 / 定时 / Webhook / Hook）、AI Agent、条件 / Switch / Merge / Filter、Set Fields、Code（多语言 + 语法检查）、Sort / Limit / Aggregate、生成图片、Loop 循环节点、子工作流、输出与格式化（模板 / JSON）、参数抽取器与问题分类器。
- dify 风格的数据流：按节点的类型化输入可绑定上游输出，变量选择器支持 `{{$nodes.*}}` 等作用域引用，配合作用域插值、env / run 变量与按节点的错误处理。
- 真实可运行：自带极简调度器（带防环），提供本地 HTTP 运行 API，支持触发器工作目录、运行历史、单节点 mock 测试、人工审批暂停节点，以及带密钥剥离的 DSL 导入 / 导出。
- 与 Kun 深度联动：工作流作为工具暴露给 Kun 智能体；可绑定到 Kun 的 hook 阶段（workflow ⇄ hook 触发器，带递归守卫）；在 code 模式聊天里集成发现与实时 UI，且无需重启 Kun 即可刷新工作流目录。
- 编辑器体验：聊天内实时运行查看面板（画布 / 列表）、按节点输入输出的运行日志面板、可主题化的缩略图与可折叠侧栏；AI 节点提供可搜索的“供应商 → 模型”选择器，生成图片节点支持按节点的供应商与图片模型。
- 自定义模块：支持保存节点预设，并可用脚本扩展自定义模块。

### 按供应商路由

- 回合可按 `thread.providerId` 路由到各自的 per-provider HTTP 客户端，多供应商混用时请求不再走错出口。

### 会话与版本控制

- 新增响应级的对话分叉（conversation forks），可以从某条回复处分岔出新的对话走向。
- 新增 Git 回合回滚检查点，按回合记录快照、按需回滚。
- 新增会话级 checkout worktree，把会话与隔离工作树绑定得更自然。

### 终端、托盘与界面

- 内置终端新增切换快捷键，并按工作区隔离会话。
- 系统托盘新增动态会话菜单，并跨平台修正项目名称的显示。
- 上下文容量表改为环形仪表；光标聚光灯覆盖更多界面；记忆按项目作用域隔离记录。
- 设置整理：权限页移到 Agents 之后、日志保留默认 3 天、合并媒体生成菜单、缩短设置中的家目录路径、保持侧栏底部可见。
- 防止不安全的视觉模型降级；保留图片附件的引用路径。

### 子代理

- 新增 over-engineering-reviewer（过度设计审查）子代理 profile。

### 本版合入的修复

- 飞书 / Telegram 等 Claw 接入：改进 Telegram 连接设置与 UI（#434）。
- 供应商：修复 GLM coding 端点并对齐压缩历史（#442）、移除已废弃的小米 flash 模型（#441）。
- composer：上下文容量随所选模型对齐（#440）、恢复 footer 选择器弹层（#438）、在项目选择器中归组 worktree 项目（#443）。
- 快捷键：macOS 终端切换改用 `Ctrl+\``（#439）。
- 侧栏：worktree 徽标移到线程标题之前（#433）。
- 预加载：避免在沙箱 preload 中引用 Node 内置模块，并加入回归守卫测试（#432）。

### 升级说明

- 从 `v0.2.14` 升级可直接通过 GUI 更新，本地数据与配置沿用，无需重建。
- 想体验工作流，可在定时任务下方找到「创建 Loop」入口；老的定时任务可以理解为工作流的单步特例。
- 如果你混用多个模型供应商，本版的按供应商路由会让请求更准地落到各自出口。

### 总结

v0.2.15 把 Kun 从“会聊天、能干活的助手”推进到“能被编排的工作流引擎”：可视化的「创建 Loop」让多步流程可画、可跑、可复用，配合按供应商路由、对话分叉和回合回滚等能力，再加上一轮扎实的修复，是 v0.2 系列里分量最重的一版。
