# DeepSeek GUI v0.2.8

这一版的重点在“多模型供应商”和“会话工作区”两条线：模型端点格式可配置、自定义供应商不再因重启丢失，会话可以直接切换工作目录，同时飞书 / Lark 接入和运行时稳定性也各有补强。

### 模型供应商与端点格式

- 支持可配置的模型端点格式，不同供应商可以走各自的 wire format。
- 修复自定义模型供应商在重启后配置丢失的问题。
- 修复部分场景下模型供应商“无反应”的问题，连接更可靠。

### 会话工作区与输入

- 支持通过 PATCH 更新会话的工作目录；当工作区切换失败时，会回退到会话选择而不是卡住。
- FloatingComposer 增强，即使还没选中运行时会话也可以先开始输入。
- 新增 composer 的访问与审阅（access / review）控制。
- 对齐了聊天工具栏与会话头部，并禁用系统级文本选中，界面更像“应用”而不是网页。

### 飞书 / Lark 接入

- 飞书 / Lark 收到消息后会先给一个“处理中”的 reaction，反馈更及时。
- 飞书 / Lark 的回包改为 Markdown 格式发送，排版更清晰。

### 运行时与稳定性

- 新增 `waitForRuntimeTurnsIdle`，等待运行时空闲的判断更准确。
- busy 超时错误会显示真实的等待时间（fixes #131）；探测失败时 runtime-hero 会在标题里显示具体错误（fixes #78）。
- 保留已完成的助手文本段落，去重重复的文件改动摘要，并恢复未完成的 SDD 草稿。
- 默认沙箱模式调整为 `danger-full-access`，并配套调整了相关组件。

### 升级说明

- 从 `v0.2.7` 升级可直接通过 GUI 更新。
- 如果你用了自定义模型供应商，升级后可以确认重启后配置是否保留；用飞书 / Lark 的话，留意回包变成 Markdown 格式。

### 总结

v0.2.8 把多供应商接入和会话工作区切换做得更稳，配合飞书 / Lark 体验和运行时空闲判断的改进，让多模型、多目录的日常使用更顺。
