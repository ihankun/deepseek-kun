# Kun v0.2.9

这是一次分量很重的版本：应用正式更名为 **Kun**，引入形象工坊和可配置 Hooks，Write 工作台切换到富文本编辑，SDD 打通“需求到开发”的可追溯闭环，同时还集中修掉了一大批 Kun 运行时的稳定性问题。它既是品牌焕新，也是运行时成熟度的一次跃升。

### 品牌焕新：Kun 与形象工坊

- 应用更名为 Kun，并加入 iKun 视觉形象。
- 新增 UI 形象工坊（UI plugin workshop），形象可以按插件方式安装与切换。
- 扩展了 Kun 的桌面端能力边界。

### 可配置 Hooks 与模型供应商

- 新增可配置的 Kun Hooks，可以在工具调用、提示等阶段插入自定义处理。
- 模型供应商支持更完整的可配置化，接入和切换更灵活。

### Write 工作台与设计稿

- Write 编辑改为 Tiptap 富文本模式（带保真度门控，可回退）。
- 新增写作原型与设计稿生成能力。
- 图片生成设置移到独立标签页；生成的图片可以跨 IM 与 Write 编辑器交付。

### SDD：需求到开发的可追溯闭环

- 通过可追溯的 R 区块，把“需求 → 计划 → 开发”串成闭环，需求和实现之间可以互相回溯。

### 性能优化

- 显著降低 Write 工作台的每次按键开销。
- 按网络分块批量发送 SSE 事件，减少跨 IPC 的开销。
- 修复启动就绪超时问题，并削减线程存储的 I/O。

### Kun 运行时稳定性

- 启动时探测 Kun `/health` 并延长就绪判定时间（#188）。
- 长时间 Agent 回合中保持微信通道响应（#191）；web_fetch 超大响应改为截断而非失败（#146）。
- busy 看门狗耗尽后解开消息队列卡死（#178）；工具执行失败以工具结果形式回传（#144）。
- 易变的 context instructions 移到历史之后以利提示缓存（#197）；内存事件总线保留量有界（#195）。
- 阻止 SSE 重放导致流式输出重复（#186 #187）；在重复“无工具”回复时停止目标续跑循环。
- 强化 plan 模式：按回合隔离工具、收尾卡住的回合项、修正用量统计。

### 升级说明

- 从 `v0.2.8` 升级可直接通过 GUI 更新；更名为 Kun 后，本地数据与配置会沿用，无需重建。
- 如果你重度使用 Write，建议体验新的富文本模式；遇到排版差异可通过保真度门控回退。

### 总结

v0.2.9 既换了“名字和脸”，也补强了“筋骨”：Kun 品牌、形象工坊、富文本写作、SDD 闭环，加上一大批运行时稳定性修复，让它在好看之外更耐用。
